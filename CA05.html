Signed Magnitude, One's Compliment, Two's Compliment

Signed Magnitude, One's Compliment, and Two's Compliment seek to solve the issue of how to represent a negative number in binary code. Each of these systems take it's own different approach to this problem.

Signed Magnitude

In signed magnitude, the leftmost bit is reserved and used as a sign bit. This sign bit determines if the binary number in question is positive or negative.
If the leftmost bit is a 1, it means the numbers is a negative. If the leftmost bit is a 0, it means the number is positive.

The range of numbers that signed magnitude can represent is dependent on the size of the binary number. N (The bit length of the binary number) can be used in a formula to calulate the range of valid numbers.
This formula is as follows. -(2N-1-1) to +(2N-1-1) where N is the bit length. Let's try an example.

Find the valid range of a signed magnitude number

-----------------------------------------------------------------------------------------------------------------------------------------------------------------> Example 1

Find the valid range of numbers for 4 bits.
We know that N (our bit length) is 4. So, we plug this into our equation 

  -(24-1-1) to +(24-1-1) 
  -(23-1) to +(23-1) 
  -(8-1) to +(8-1) 
  -7 to +7

So, our valid range of numbers in signed magnitude for bit lenth 4 is -7 to +7.
If we have a 4 bit binary number and we use 1 bit for the sign, we can only use 3 to hold value.

Find a particular binary number in signed magnitude

Finding a number in signed magnitude is easy. To do this, we first find the normal binary representation, then add on a sign bit.
We add on a 1 if the number is negative, and a 0 if the number is positive. 

-----------------------------------------------------------------------------------------------------------------------------------------------------------------> Example 1

Find the signed magnitude representation of -20
Our first step is to find the normal binary representation for the number 20. 

 0   0   0   0   0 
 ----------------- 
 16  8   4   2   1

We ask ourselves, which combination would give us 20? Well, 16 + 4 would, so we put a 1 in those columns

 1   0   1   0   0 
 ----------------- 
 16  8   4   2   1

After doing this, we see that 20 in binary is 101002
Now, all we have to do is add a 1 or a 0, depending on if we're trying to find the positive or negative version of 20.
Since we are trying to find the negative version, we add on a 1 for our sign bit. So, we get 

 1 + 10100

-20 in signed magnitude is 1101002 

-----------------------------------------------------------------------------------------------------------------------------------------------------------------> Example 2

Find the signed magnitude representation of +20
From our last example, we know that 20 in binary is 101002

 1   0   1   0   0 
 ----------------- 
 16  8   4   2   1

Now, we just add on a 0 for the sign bit because we are trying to find the positive version of 20. 

 0 + 10100

+20 in signed magnitude is 0101002
