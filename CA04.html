Radix Forms

Radix forms are a way of representing numbers that are in a specific number system. Binary, Decimal, and Octal can all be represented in a radix form.
We can represent these forms of numbers by using the base of the number system raised to the position of the number, and then multiplying that by the corrisponding value in the original number. We then add all of our numbers together.

Binary Radix Representation

-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------> Example 1

Consider the binary number 110012
To go about representing this as a radix form, we multiply each number by the base raised to the position of the number. We then add everything up.
Let's first determine the positions of the numbers.

Note: Since we are using a base 2 number system (Binary) we use base 2 and raise it to the position of the numbers. 

 1   1   0   0   1 
 ----------------- 
 24  23  22  21  20

Now that we have determined the positions of this binary number, all that is left to do is multiply the numbers by the matching binary number.
Starting from the left, our first part of the radix form is 

 1 x 24

This is because we have a 1 that corrisponds to the 24 position. Next, we add 

 1 x 24 + 1 x 23

Because we see a 1 is corrisponding to the 23 position, we multiply by 1. We then do 

 1 x 24 + 1 x 23 + 0 x 22

We see that a 0 is corrisponding to the 22 position. So, we multiply by 0. After this, we do 

 1 x 24 + 1 x 23 + 0 x 22 + 0 x 21

Since a 0 corrisponds to 21, we multiply that by 0. Finally, we have 

 1 x 24 + 1 x 23 + 0 x 22 + 0 x 21 + 1 x 20

1 corrisponds to 20, so we multiply by 1.
Now that we have gone through each position, we are done!
110012 represented as a radix form is 

 1 x 24 + 1 x 23 + 0 x 22 + 0 x 21 + 1 x 20

-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------> Example 2

Consider the binary number 111.011
To represent this as a radix form, we first should determine the positions of the numbers. 

 1   1   1   .   0   1   1 
 ------------------------- 
 22  21  20     2-1  2-2  2-3

After this, we simply multiply by 1 or 0 depending on if we see a 1 or 0 in the original binary number. After that, we add everything. This gives us 

 1 x 22 + 1 x 21 + 1 x 20 + 0 x 2-1 + 1 x 2-2 + 1 x 2-3

So, 111.011 represented as a radix form is 1 x 22 + 1 x 21 + 1 x 20 + 0 x 2-1 + 1 x 2-2 + 1 x 2-3

Decimal Radix Representation

-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------> Example 1

Consider the decimal number 432.110
Since we are using a decimal number, we use base 10, and raise it to the position of the numbers. Let's first determine positions. 

  4   3   2   .   1 
 ----------------- 
 102 101 100    10-1

Notice how we are using base 10. Our exponents remain the same, but since this is a decimal number, we use base 10 instead of base 2 like when we did a binary number.
Now, all that we need to do is multiply the position with the corrisponding number. After we do this, we add everything together. 

 4 x 102 + 3 x 101 + 2 x 100 + 1 x 10-1

So, 432.110 represented as a radix form is 4 x 102 + 3 x 101 + 2 x 100 + 1 x 10-1

-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------> Example 2

Consider the decimal number 999.4410
Before we start, let's determine the positions of the numbers. Remember, we use base 10 since we are using the decimal number system.

  9   9   9   .   4   4 
 ---------------------- 
 102 101 100     10-1 10-2

We then multiply by the number we see that corrisponds. After that, we add everything together. 

 9 x 102 + 9 x 101 + 9 x 100 + 4 x 10-1 + 4 x 10-2

So, 999.4410 as a radix form is 9 x 102 + 9 x 101 + 9 x 100 + 4 x 10-1 + 4 x 10-2

When representing a hexadecimal number as a radix form, sometimes we must convert our characters into numbers.
For example, you are unable to use the letter D in a radix form. You must convert this letter into the number 13, since our table tells us that D = 13

-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------> Example 1

Consider the number 4AB.D116
We first determine which position each number/character is in. Make sure to use a base of 16 for positional numbers, since we are representing a hexadecimal number. 

 4   A   B   .   D   1 
 --------------------- 
162 161 160     16-1 16-2

Now, all that remains to be done is to multiply each value in our original hexadecimal number by the corrisponding value. Then, we add everything together. So, we do 

 4 x 162 + A x 161 + B x 160 + D x 16-1 + 1 x 16-2

However, when converting hexadecimal to a radix form, we cannot have letter characters such as A, B, and D. So, we convert these values from hexadecimal to decimal using our conversion table.
Looking at the table above, we can see that 

 A = 10 
 B = 11 
 D = 13

Now that we have found the decimal values, all we do is substitute them into the radix form we found prior. 

 4 x 162 + A x 161 + B x 160 + D x 16-1 + 1 x 16-2 <-- Before Substitution 
 4 x 162 + 10 x 161 + 11 x 160 + 13 x 16-1 + 1 x 16-2 <-- After Substitution

So, 4AB.D116 as a radix form is 4 x 162 + 10 x 161 + 11 x 160 + 13 x 16-1 + 1 x 16-2

-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------> Example 2

Consider the number 1F.0A16
Our first step is to determine the positions of the characters/numbers. 

 1   F   .   0   A 
 ------------------ 
 161 160    16-1 16-2

Next, we multiply each value in our original number by the corrisponding value. Then, we add everything together. 

 1 x 161 + F x 160 + 0 x 16-1 + A x 16-2

However, we notice that we have letters in our radix form. We cannot have these, so we convert these letters from hexadecimal to decimal. 

 F = 15 
 A = 10

Now that we have converted our letter values from hexadecimal to decimal, we can then substitute in. 

 1 x 161 + F x 160 + 0 x 16-1 + A x 16-2 <-- Before Substitution
 1 x 161 + 15 x 160 + 0 x 16-1 + 10 x 16-2 <-- After Substitution

So, 1F.0A16 as a radix form is 1 x 161 + 15 x 160 + 0 x 16-1 + 10 x 16-2
